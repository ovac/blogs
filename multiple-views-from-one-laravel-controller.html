Different views from one Laravel controller method.

Interestingly, this is very simple, but it could keep you hitting your head for a few hours.

When faced with this challenge, the recommended option is to implement the methods required on a base controller, and then for each view that needs the data on the base controller, a new controller should be created.   Route::resource('{path?}', 'HelloController')
